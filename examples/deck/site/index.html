How a write flows

Governed by construction.

An operator action is a policy-checked, audited, reversible transaction — not a raw SQL update.

# northwind policy — refund_queue.app
write refunds:
  table:    finance.refunds
  who:      role in [ops_lead, finance]      # SSO group
  guard:    amount <= 500 or approval(finance_manager)
  audit:    actor, before, after, reason     # always on
  rollback: enabled                          # one click, any version
Operator→ Policy check→ Approval→ Warehouse write→ Audit + version
This is the technical credibility slide. Engineers in the room relax here: it's not magic, it's a typed policy over a transaction. Mention SOC 2 Type II in progress if a security-minded partner asks.
