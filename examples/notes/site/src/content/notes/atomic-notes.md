An **atomic note** holds exactly one idea. Not one topic, not one source — one
claim, small enough that you could link to it from a precise spot in your thinking.

The test I use: *can I link to this without linking to the wrong thing?* If a note
is about both the forgetting curve **and** flashcard scheduling, I can't link
cleanly to either, so it wants to be split. Splitting feels like extra work and is
secretly the work — the seams between ideas are where the interesting connections
live.

Atomicity is what makes everything else in the garden possible:

- It makes [[Evergreen notes]] reusable, because a single idea travels across
  projects in a way a sprawling page never does.
- It makes the [[Zettelkasten]] method run — the whole technique assumes each slip
  carries one thought.
- It makes links *mean* something. A link between two atomic notes is a real
  claim about how two ideas relate; a link between two grab-bag pages is noise.

Where I still get it wrong: I under-split. A note quietly grows three ideas while
I'm not looking, and I only notice when I try to link to it and can't. Catching that
drift is half of what [[Tending notes]] is for.
