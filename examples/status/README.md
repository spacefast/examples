# Northwind API — System Status

A public status page for a developer API product, built as a self-contained static site (`index.html` + `styles.css` + `app.js`, no build step). It opens with an overall-status banner whose color and wording are derived in JavaScript from the worst component state, alongside a live "last updated" timestamp and a 90-day uptime headline. A components list (API, Dashboard, Webhooks, CDN, Database) renders a colored status pill and a 90-day uptime bar for each subsystem — 90 deterministic, tooltipped cells with a computed uptime percentage — and an active-incident block walks a newest-first update timeline through the standard Investigating → Identified → Monitoring → Resolved lifecycle. Below it sit scheduled-maintenance windows, a date-grouped past-incident history with severity pills and durations, and a subscribe box that validates the email, shows an inline error, and swaps to a persisted "you're subscribed" confirmation via `localStorage`. The look is a calm dark-navy NOC dashboard with a green operational accent (amber/red/indigo for the other states), Inter for text and JetBrains Mono for figures, fully responsive and accessible. See `prompt.md` to build your own version and `site/` for the finished example.
